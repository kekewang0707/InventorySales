# v0.2.0 — 销售出库 + 送货单打印

## 目标
用户根据生产情况填写送货单，并能连接打印机打印送货单。

---

## 1) 数据模型

### DeliveryNote（送货单主表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | 自增主键 |
| doc_number | String(50) | 送货单编号（自动生成 `DH-20260607-001`） |
| customer_id | Integer FK → Customer | 客户 |
| delivery_date | Date | 送货日期 |
| total_amount | Decimal(12,2) | 总金额（由明细行计算） |
| remark | Text | 备注 |
| status | String(20) | 状态：`draft` → `saved` → `reviewed` |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | |

### DeliveryNoteItem（送货单明细行）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | |
| delivery_note_id | Integer FK → DeliveryNote | 所属送货单 |
| product_id | Integer FK → Product | 产品 |
| quantity | Decimal(10,2) | 数量 |
| unit_price | Decimal(10,2) | 单价（自动填充 Product.default_price，允许手动覆盖） |
| subtotal | Decimal(12,2) | 小计（quantity × unit_price） |
| remark | Text | 行备注 |

---

## 2) API 端点

- `GET /api/delivery-notes` — 分页列表（支持按客户/日期范围搜索）
- `POST /api/delivery-notes` — 创建送货单（含明细行）
- `GET /api/delivery-notes/{id}` — 查看单张送货单详情
- `PUT /api/delivery-notes/{id}` — 编辑（整体替换明细行）
- `DELETE /api/delivery-notes/{id}` — 删除
- `PUT /api/delivery-notes/{id}/status` — 推进状态（`draft` → `saved` → `reviewed`）
- `PUT /api/delivery-notes/{id}/status-revert` — 回退状态（`reviewed` → `saved` → `draft`）
- `POST /api/delivery-notes/{id}/print` — 生成 PDF 并返回文件路径

---

## 3) PDF 送货单

**技术选型**: reportlab

**模板内容**（A4 纸张）:
- 抬头：公司名称 + "送货单"
- 单据编号、日期、客户信息
- 表格：序号 | 产品名称 | 规格 | 数量 | 单价 | 小计 | 备注
- 底部：合计金额（大写 + 小写）、制单人、客户签收
- 页码

**中文字体**: 内置思源黑体到 `backend/assets/fonts/`

---

## 4) 打印流程

```
前端点击"打印"
  → POST /api/delivery-notes/{id}/print
  → 后端生成 PDF 临时文件
  → 返回 PDF 文件路径
  → PyQt6 print_handler.py 接收 PDF
  → QPrinter 打开系统打印对话框
  → 用户选择打印机 → 打印
```

## 5) Vue 前端页面

- **送货单列表页**: El-Table，搜索按客户+日期范围
- **送货单表单页**: 客户选择器、日期、备注；明细行动态增删，自动计算小计
- **送货单详情页**: 只读视图 + "打印" 按钮
- **导航**: 左侧菜单新增 "送货单管理"

---

## 6) 删除约束

- 删除产品前，检查是否被 DeliveryNoteItem 引用；有引用则返回错误提示，禁止删除
- 删除客户前，检查是否被 DeliveryNote 引用；有引用则返回错误提示，禁止删除
