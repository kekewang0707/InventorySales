# v0.1.1 — 日志审计 + Excel 导入

## 目标
- 记录所有产品/客户的数据变更历史，可追溯
- 支持通过 Excel 批量导入产品和客户数据

---

## 1) 日志审计（Audit Log）

### 数据模型 — AuditLog

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer PK | 自增主键 |
| entity_type | String(50) | 操作对象类型：`product` / `customer` |
| entity_id | Integer | 操作对象 ID |
| action | String(20) | 操作：`create` / `update` / `delete` / `import` |
| old_values | Text(JSON) | 变更前数据（JSON，delete 为完整记录） |
| new_values | Text(JSON) | 变更后数据（JSON，create 为完整记录） |
| operator | String(100) | 操作人（单用户系统固定为 `system`） |
| created_at | DateTime | 操作时间 |

### 后端新增

| 文件 | 用途 |
|------|------|
| `backend/models/audit_log.py` | AuditLog 模型 |
| `backend/schemas/audit_log.py` | Pydantic 模式 |
| `backend/services/audit_service.py` | 写入 + 分页查询 |
| `backend/routers/audit_logs.py` | `GET /api/audit-logs` 分页查询 |

**`audit_service.py` 接口：**
- `log_create(db, entity_type, entity_id, new_values, operator="system")`
- `log_update(db, entity_type, entity_id, old_values, new_values, operator="system")`
- `log_delete(db, entity_type, entity_id, old_values, operator="system")`
- `list_logs(db, entity_type, action, page, page_size) -> (total, items)`

### 后端修改

- `backend/services/product_service.py` — 每个 CRUD 操作末尾调用 audit log
- `backend/services/customer_service.py` — 每个 CRUD 操作末尾调用 audit log
- `backend/main.py` — 注册 `audit_logs` 路由

### 前端页面

- `frontend/src/views/logs/AuditLogList.vue`
  - El-Table：时间 | 对象类型 | 操作 | 对象 ID
  - 行展开查看 old/new 详细 JSON
  - 顶部筛选：对象类型下拉、操作类型下拉
- `frontend/src/components/AppLayout.vue` — 左侧菜单新增 "操作日志"
- `frontend/src/router/index.js` — 新增 `/logs` 路由

## 2) Excel 导入

### 技术选型
- `openpyxl` 解析 .xlsx
- 后端生成模板文件（无需额外依赖）

### 可下载模板

服务端动态生成 .xlsx 文件，提供两个端点：
- `GET /api/import/template?entity_type=product` — 下载产品导入模板
- `GET /api/import/template?entity_type=customer` — 下载客户导入模板

**产品模板表头：** 产品名称（必填）、规格型号、默认单价、备注
**客户模板表头：** 客户名称（必填）、联系人、联系电话（必填）、地址、备注

### 导入端点

- `POST /api/import/preview` — 上传 .xlsx 预览
  - multipart/form-data：`file` + `entity_type`
  - 返回：`{ total_rows, valid_rows, errors: [{row, message}], preview: [...] }`
- `POST /api/import/confirm` — 确认导入
  - Request: `{ entity_type, session_id }`
  - 返回：`{ imported, failed, errors: [...] }`
- 导入操作也记录 audit log（action=`import`）

### 文件

| 文件 | 用途 |
|------|------|
| `backend/services/excel_service.py` | 模板生成 + 文件解析 + 校验 + 导入 |
| `backend/routers/imports.py` | 导入路由 |
| `frontend/src/components/ImportDialog.vue` | 导入对话框 |

### 前端

- **ImportDialog.vue** 通用导入组件
  - 对象类型选择器
  - "下载模板" 按钮
  - 文件选择器（.xlsx）
  - 上传后预览表格 + 错误行标记
  - 确认导入按钮
- `ProductList.vue` 和 `CustomerList.vue` 各加一个 "导入" 按钮
