# v1.1.0 — AI 多轮对话记忆

## 目标

在 v1.0.0 AI 快捷操作的基础上，增加**跨轮次对话历史记忆**。用户可以连续对话：问"上个月的送货单" → AI 回复后 → 追问"金额最高的那笔" → AI 能理解"那笔"指代上一条回复中的送货单，无需每次都完整重复指令。

v1.0.0 中每次 `POST /api/ai/command` 都是独立请求，后端完全不记得上一轮说了什么。前端虽然展示了消息列表，但那只是纯 UI 缓存，对 LLM 推理没有贡献。

---

## 1) 设计思路

### 核心方案：服务端会话（Session）存储对话历史

```
用户: "查一下上个月的送货单"
  → POST /api/ai/command { text: "...", session_id: "abc123" }
  → 后端找到 session abc123，追加用户消息
  → LLM 看到完整对话历史 → 回复 + 追加到 session
  → 返回 reply + session_id

用户: "金额最高的那笔是多少？"
  → POST /api/ai/command { text: "...", session_id: "abc123" }
  → 后端找到 session abc123，看到上一轮历史
  → LLM 理解"那笔"指的是上个月的送货单 → 回复
  → 返回 reply + session_id
```

### 为什么不在前端传全部历史？

- 前端只存纯文本展示消息，不存工具调用中间结果
- 后端 ReAct 循环需要在历史中保留 **Tool Call → Observation** 的完整链条，前端拿不到 Observation
- 服务端会话可以实现会话数上限、单会话消息数上限、淘汰等管理
- 后续扩展（持久化到数据库、多用户隔离）更方便

## 2) 会话 Session 管理

### 数据结构

```python
@dataclass
class SessionMessage:
    role: str       # "user" | "assistant" | "tool_call" | "tool_result"
    content: str    # 消息文本 / 工具调用描述 / 工具返回结果

@dataclass
class ConversationSession:
    session_id: str
    messages: list[SessionMessage]   # 完整消息列表
    created_at: datetime
    last_active: datetime            # 每次活动更新
    title: str = ""                   # 自定义会话标题（前端重命名）
```

### 会话配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 最大消息数（单会话） | 30 条 | 超过时裁剪最早的消息（保留系统提示空间） |
| 最大会话数（全局） | 30 个 | 超过时淘汰最久未活跃的会话 |
| 淘汰策略 | 每次创建新会话时 | 淘汰 `last_active` 最早的会话 |

> 会话不设时间 TTL，持久化到 SQLite 数据库，应用重启后自动恢复。

### 性能预期

| 操作 | 预期延迟 | 说明 |
|------|---------|------|
| 获取会话 | < 1ms | 纯内存 dict 查找 |
| 追加消息 | < 5ms | 内存追加 + SQLite INSERT（WAL 模式，读写不互斥） |
| 启动恢复 | < 100ms | 加载最多 30 个会话及其消息 |
| 定时清理 | < 50ms | 每 6 小时执行一次，删除 7 天前的会话 |

### Session ID 生命周期

- **浏览器 Tab 级别**：前端在 Tab 生命周期内生成一个 UUID 作为 session_id
- **关闭对话框不销毁**：关闭 AI 浮窗再打开，同一 Tab 内 session 延续
- **刷新页面 / 打开新 Tab**：生成新 session_id，旧 session 保留在服务端（最多 30 个），直到被新会话淘汰
### 会话存储

### 数据库模型

利用项目已有的 SQLAlchemy + SQLite 基础设施，新增两张表持久化会话数据。

```python
# backend/models/ai_session.py
from datetime import datetime
from sqlalchemy import String, Text, DateTime, ForeignKey, Integer
from sqlalchemy.orm import Mapped, mapped_column, relationship
from backend.database import Base


class AiSession(Base):
    __tablename__ = "ai_sessions"

    session_id: Mapped[str] = mapped_column(String(64), primary_key=True)
    created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.now)
    last_active: Mapped[datetime] = mapped_column(DateTime, default=datetime.now, onupdate=datetime.now)

    messages: Mapped[list["AiSessionMessage"]] = relationship(
        back_populates="session", cascade="all, delete-orphan",
        order_by="AiSessionMessage.seq"
    )


class AiSessionMessage(Base):
    __tablename__ = "ai_session_messages"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    session_id: Mapped[str] = mapped_column(ForeignKey("ai_sessions.session_id", ondelete="CASCADE"))
    seq: Mapped[int] = mapped_column(Integer, nullable=False, comment="消息序号，按序排列")
    role: Mapped[str] = mapped_column(String(16), nullable=False, comment="user / assistant / tool_call / tool_result")
    content: Mapped[str] = mapped_column(Text, nullable=False)
    created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.now)

    session: Mapped["AiSession"] = relationship(back_populates="messages")
```

### 存储架构：内存缓存 + 数据库持久化

```
请求时读写
  ┌──────────────┐    实时写入    ┌──────────┐
  │ _sessions    │ ────────────→ │ SQLite   │
  │ (内存 dict)  │ ←──────────── │ (持久化) │
  └──────────────┘  启动时加载    └──────────┘
```

- **运行期**：读写都在内存 `_sessions` dict 上，保证请求延迟与纯内存方案一致
- **写穿透**：每次 `_append_message()` 将新消息写入内存后，同步写一条记录到 `ai_session_messages` 表；新会话创建时写入 `ai_sessions` 表
- **启动恢复**：`ai_service.py` 提供 `load_sessions_from_db(db)` 函数，应用启动时由 `main.py` 调用，从 DB 加载所有会话到内存
- **淘汰时同步删除**：`_evict_sessions_if_needed()` 在从内存删除会话后，同步从 DB 删除对应行（`ai_session` 的 cascade 自动级联删除 messages）

### 并发安全

单用户系统仍可能有并发请求（快速连续提问、前端重试等）。为每个 session 分配一把锁，确保消息顺序。

```python
_session_locks: Dict[str, asyncio.Lock] = {}

def _get_session_lock(session_id: str) -> asyncio.Lock:
    """获取 session 级别的锁（lazy 初始化）。"""
    if session_id not in _session_locks:
        _session_locks[session_id] = asyncio.Lock()
    return _session_locks[session_id]

def _cleanup_orphan_locks():
    """移除已不存在会话的锁对象，防止内存泄漏。"""
    dead = [sid for sid in _session_locks if sid not in _sessions]
    for sid in dead:
        _session_locks.pop(sid, None)
```

在 `handle_command` 和 `execute_pending` 的 session 操作段使用 `async with`：

```python
# handle_command 中的使用模式
sid, session = await _get_or_create_session(db, session_id)
lock = _get_session_lock(sid)
async with lock:
    await _append_message(db, session, SessionMessage(role="user", content=text))
    # ... 后续 LLM 调用和 AI 回复写入 ...
```

### 错误重试

写入 DB 可能因临时锁冲突失败，加指数退避重试：

```python
async def _append_message_with_retry(db, session: ConversationSession, msg: SessionMessage,
                                      max_retries: int = 3):
    """带重试的消息追加，内存失败直接抛出，DB 失败最多重试 3 次。"""
    # 先写入内存（同步，不会失败）
    session.messages.append(msg)
    session.last_active = datetime.now()

    # 再写入 DB（异步，可能临时失败）
    last_exc = None
    for attempt in range(max_retries):
        try:
            seq = len(session.messages)
            db_msg = AiSessionMessage(
                session_id=session.session_id, seq=seq, role=msg.role, content=msg.content
            )
            db.add(db_msg)
            await db.commit()
            return
        except Exception as e:
            last_exc = e
            await db.rollback()
            if attempt < max_retries - 1:
                await asyncio.sleep(0.1 * (attempt + 1))  # 退避：0.1s, 0.2s, 0.3s
                continue
    # 全部重试失败：从内存回滚
    session.messages.pop()
    raise last_exc
```

### 定时清理

除创建新会话时淘汰外，额外提供一个定时清理任务，删除超过 7 天未活跃的会话：

```python
async def cleanup_stale_sessions(db, max_age_hours: int = 168):
    """清理长时间未活动的会话（默认 7 天）。"""
    cutoff = datetime.now() - timedelta(hours=max_age_hours)
    stale = [sid for sid, s in _sessions.items() if s.last_active < cutoff]
    if not stale:
        return
    await db.execute(delete(AiSession).where(AiSession.session_id.in_(stale)))
    await db.commit()
    for sid in stale:
        _sessions.pop(sid, None)
    logger.info("Cleaned up %d stale session(s) older than %dh", len(stale), max_age_hours)
```

`main.py` 中注册定时任务：

```python
async def scheduled_cleanup():
    while True:
        await asyncio.sleep(3600 * 6)  # 每 6 小时执行一次
        try:
            async with async_session_factory() as db:
                from backend.services.ai_service import cleanup_stale_sessions
                await cleanup_stale_sessions(db)
        except Exception as e:
            logger.warning("Session cleanup failed: %s", e)
```

## 3) Prompt 改造

### 当前 v1.0.0 的 Prompt（无历史）

```
你是一个具备推理和行动能力的AI助手。
...
## 当前任务
**Question:** {question}

## 执行历史
{history}
```

### v1.1.0 增加对话历史注入

在 `## 当前任务` 之前插入 `## 对话历史` 区块：

```
## 对话历史
user: 查一下上个月的送货单
assistant: 找到了 5 张送货单，合计 32,850 元...
  （其中调用了 list_delivery_notes → 返回结果）

## 当前任务
**Question:** 金额最高的那笔是多少？

## 执行历史
（本轮的 ReAct 循环记录，初始为空）
```

### 历史消息的格式控制

- **用户消息** → `user: {content}`
- **AI 回复（Finish）** → `assistant: {content}`
- **工具调用（确认前）** → 不单独记录工具调用详情到对话历史，只记录最终 finish 回复
- **工具调用（确认后执行成功）** → `assistant: {content}` 其中包含执行结果
- **Observation** 不在对话历史中保留（仅保留在当前轮次的 `history` 变量中）

### 消息裁剪策略

当 `messages` 超过 `_MAX_SESSION_MESSAGES`（30 条）时：
- 保留最新的 25 条完整轮次
- 保留最早 1 条作为上下文锚点（通常是第一条用户消息 + 回复）
- 裁剪中间部分，并在对话历史开头追加 `...（省略了 N 条历史消息）...`

```
[前端] 生成 session_id（Tab 级别）

  1. POST /api/ai/command { text, session_id }
       │
       ▼
  2. ai_service.handle_command()
       │

       ├─ 追加用户消息到 session.messages
       ├─ 构造含对话历史的 prompt

       │
       ├→ 查询类 → 追加 AI reply 到 session → 返回
       │
       └→ 写入类 → 暂存 pending（含 session_id）
                  → 返回 needs_confirm + session_id

  3. 用户确认
       │
       ▼
  4. POST /api/ai/confirm { confirm_id }
       │
       ├─ 执行写入
       ├─ 将结果追加到 session.messages
       └─ 返回 executed + session_id
```

## 7) 关键实现逻辑

### 7.1 获取/创建会话

```python
def _get_or_create_session(session_id: str = "") -> tuple[str, ConversationSession]:
    """根据 session_id 获取已有会话，或创建新会话。返回 (session_id, session)。"""
    await _evict_sessions_if_needed(db)
    if session_id and session_id in _sessions:
        session = _sessions[session_id]
        session.last_active = datetime.now()
        return session_id, session
    # 创建新会话（写内存 + 写 DB）
    new_id = uuid.uuid4().hex
    session = ConversationSession(
        session_id=new_id,
        messages=[],
        created_at=datetime.now(),
        last_active=datetime.now(),
    )
    _sessions[new_id] = session
    db.add(AiSession(session_id=new_id, created_at=session.created_at, last_active=session.last_active))
    await db.commit()
    return new_id, session
```

### 7.2 构建含历史的 Prompt

```python
def _build_llm_prompt_with_history(question: str, session: ConversationSession,
                                    tools_desc: str) -> str:
    """构建包含对话历史的 LLM prompt。"""
    history_lines = []
    for msg in session.messages:
        if msg.role == "user":
            history_lines.append(f"user: {msg.content}")

            history_lines.append(f"assistant: {msg.content}")

    history_text = ""
    if history_lines:

    return DEFAULT_REACT_PROMPT.format(
        tools=tools_desc,
        question=question,
        history="（新的一轮对话，初始无执行历史）" + history_text,
    )
```

### 7.3 `handle_command` 改造

```python
async def handle_command(text: str, db, session_id: str = "") -> dict:
    text = text.strip()
    if not text:
        return {"action": "error", "reply": "请输入指令", "confirm_id": None,
                "tool_calls": None, "session_id": session_id}

    # 获取/创建会话
    sid, session = await _get_or_create_session(db, session_id)
    session.last_active = datetime.now()

    # 追加用户消息（写内存 + 写 DB）
    await _append_message(db, session, SessionMessage(role="user", content=text))

    if not is_openai_configured():
        result = await _try_offline(text, db)
        result["session_id"] = sid
        if result["action"] == "queried":
            await _append_message(db, session, SessionMessage(role="assistant", content=result["reply"]))
        return result

    try:
        result = await _run_react_loop(text, db, session)
        result["session_id"] = sid

        if result["action"] == "queried":
            await _append_message(db, session, SessionMessage(role="assistant", content=result["reply"]))
        elif result["action"] == "needs_confirm":
            # 找到对应的 PendingAction，记录 session_id
            cid = result.get("confirm_id")
            if cid and cid in _pending_actions:
                _pending_actions[cid].session_id = sid

        return result
    except Exception as e:
        logger.warning("ReAct loop failed, falling back to offline: %s", e)
        result = await _try_offline(text, db)
        result["session_id"] = sid
        if result["action"] == "queried":
            await _append_message(db, session, SessionMessage(role="assistant", content=result["reply"]))
        return result
```

### 7.4 `execute_pending` 改造

### 7.5 消息裁剪

```python
async def _append_message_with_retry(db, session: ConversationSession, msg: SessionMessage,
                                      max_retries: int = 3):
    """带重试的消息追加（写内存 + 写 DB），超过上限时裁剪。"""
    session.messages.append(msg)
    session.last_active = datetime.now()

    # 写 DB（带指数退避重试）
    last_exc = None
    for attempt in range(max_retries):
        try:
            seq = len(session.messages)
            db_msg = AiSessionMessage(session_id=session.session_id, seq=seq,
                                      role=msg.role, content=msg.content)
            db.add(db_msg)
            await db.commit()
            break
        except Exception as e:
            last_exc = e
            await db.rollback()
            if attempt < max_retries - 1:
                await asyncio.sleep(0.1 * (attempt + 1))
                continue
            session.messages.pop()
            raise last_exc

    # 消息裁剪
    if len(session.messages) > _MAX_SESSION_MESSAGES:
        trimmed_count = len(session.messages) - _MAX_SESSION_MESSAGES + 1
        first = session.messages[0]
        keep_from = trimmed_count + 1
        session.messages = [first] + session.messages[keep_from:]
        session.messages.insert(1, SessionMessage(
            role="assistant",
            content=f"...（省略了 {trimmed_count} 条历史消息）..."
        ))
```

### 7.6 会话淘汰（超出上限时）

```python
async def _evict_sessions_if_needed(db):
    """超出 _MAX_SESSIONS 上限时，淘汰 last_active 最早的会话（内存 + DB 同步删除）。"""
    if len(_sessions) <= _MAX_SESSIONS:
        return
    sorted_sessions = sorted(_sessions.items(), key=lambda kv: kv[1].last_active)
    evict_count = len(_sessions) - _MAX_SESSIONS
    to_evict = [sid for sid, _ in sorted_sessions[:evict_count]]
    for sid in to_evict:
        _sessions.pop(sid, None)
    if to_evict:
        await db.execute(delete(AiSession).where(AiSession.session_id.in_(to_evict)))
        await db.commit()
        _cleanup_orphan_locks()
        logger.info("Evicted %d stale session(s), remaining: %d", evict_count, len(_sessions))
```

### 7.7 启动恢复

应用重启时，从 SQLite 加载所有会话到内存缓存。

```python
async def load_sessions_from_db(db):
    """从 DB 加载所有会话到内存 _sessions，在应用 startup 时调用（最多 _MAX_SESSIONS 个）。"""
    rows = await db.execute(
        select(AiSession).order_by(AiSession.last_active.desc()).limit(_MAX_SESSIONS)
    )
    sessions = rows.scalars().all()
    for s in sessions:
        msg_rows = await db.execute(
            select(AiSessionMessage)
            .where(AiSessionMessage.session_id == s.session_id)
            .order_by(AiSessionMessage.seq)
        )
        msgs = msg_rows.scalars().all()
        session = ConversationSession(
            session_id=s.session_id,
            messages=[SessionMessage(role=m.role, content=m.content) for m in msgs],
            created_at=s.created_at,
            last_active=s.last_active,
            title=s.title or "",
        )
        _sessions[s.session_id] = session
    logger.info("Loaded %d AI sessions from DB", len(sessions))
```

`main.py` 的 startup 事件中调用：

```python
# backend/main.py
@app.on_event("startup")
async def startup():
    await init_db()
    async with async_session_factory() as db:
        from backend.services.ai_service import load_sessions_from_db
        await load_sessions_from_db(db)
    asyncio.create_task(scheduled_cleanup())  # 启动定时清理

async def scheduled_cleanup():
    """每 6 小时清理一次超过 7 天未活跃的会话。"""
    while True:
        await asyncio.sleep(3600 * 6)
        try:
            async with async_session_factory() as db:
                from backend.services.ai_service import cleanup_stale_sessions
                await cleanup_stale_sessions(db)
        except Exception as e:
            logger.warning("Session cleanup failed: %s", e)
```

---

`_sessions` 自身不设时间 TTL。会话一旦创建就一直保留，直到被新会话淘汰出上限（LRU 风格）。这意味着一个 Tab 长期不使用也不会自动消失，但 30 个并发的上限对单用户系统来说绰绰有余。

## 10) 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 用户输入空文本 | 返回 error，session 不追加空消息 |
| session_id 不存在 | 后台自动创建新 session（可能淘汰最久未活跃的旧 session），返回新 session_id |
| 对话超过 30 轮 | 裁剪最早消息（保留首条），插入省略标记 |
| LLM 调用失败，降级到离线 | 离线模式保持 session 写入，确保历史不中断 |
| 用户确认写入后 | 将执行结果作为 AI 回复追加到 session |
| 用户连续两次快速提问 | 两次请求独立处理，session 按顺序追加 |
| 对话框关闭再打开（未刷新） | 前端保留本地 messages + session_id，无缝继续 |
| 浏览器刷新 | 前端生成新 session_id；`loadExistingSessions()` 从后端恢复所有会话及其消息；自定义标题从 `s.title` 恢复 |
| 会话重命名后重启 | 后端 `rename_session` 将标题持久化到 DB，重启后 `load_sessions_from_db` 加载 title，前端优先使用 `s.title` |
| 删除会话后重启 | `delete_session` 删除内存 + DB，重启后不会重新出现（顺序：先删 DB 后删内存） |

## 13) 风险与注意事项

### 内存与 DB 不一致

`_append_message_with_retry` 先写内存、再写 DB。若 DB 重试全部失败，会从内存 pop 回滚。但仍存在以下时序窗口：

```
内存已追加 msg_3 → DB 写入失败 → 回滚成功 → 一致
内存已追加 msg_3 → 应用崩溃    → 重启后从 DB 恢复 → msg_3 丢失 → 可接受
```

数据一致性要求不严格（纯对话历史），此窗口可接受。

### `_session_locks` 内存泄漏

每创建一个 session 就对应一把锁。已在 `_evict_sessions_if_needed` 淘汰会话时同步调用 `_cleanup_orphan_locks()` 清理。另外每 60 次操作触发一次惰性清理：

```python
_op_counter = 0

def _maybe_cleanup_locks():
    global _op_counter
    _op_counter += 1
    if _op_counter % _SESSION_LOCK_CLEANUP_INTERVAL == 0:
        _cleanup_orphan_locks()
```

### 前端 session_id 丢失

Tab 刷新后前端丢失 session_id，生成新 ID。用户可能困惑"刚才的对话怎么没了"。当前方案依赖服务端持久化 + 前端保留本地 messages 数组：

- **Tab 刷新 → 前端消息清空，服务端 session 保留**。用户可继续用旧 session_id 提问（需 URL 参数或 localStorage 保存 session_id）
- **v1.1.0 暂不实现 localStorage 持久化**，后续版本可考虑将 `tabSessionId` 写入 `sessionStorage`（Tab 内跨页面，刷新后消失）或 `localStorage`（跨 Tab/跨重启）

### 定时清理 vs 用户预期

7 天不活跃的会话自动删除。如果用户有隔月使用的场景，需要将 `_STALE_SESSION_AGE_HOURS` 调大或提供配置入口。v1.1.0 固定在 168 小时（7 天），后续版本可加入配置项。

### 定时清理 vs 全局上限

两个淘汰机制协同：
- **全局上限（30 个）**：任何时候 `_sessions` 不超过 30 个，超出时淘汰最久未活跃的
- **定时清理（7 天）**：后台任务删除很久不用的会话，为活跃会话腾出空间
- 两者不冲突：定时清理减少了全局上限被触发的概率
